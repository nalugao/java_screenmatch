# ScreenMatch

Um projeto de gerenciamento de filmes e séries que demonstra a aplicação de conceitos fundamentais de Programação Orientada a Objetos (POO) em Java.

## Conceitos de POO Implementados

### Encapsulamento
O projeto implementa o encapsulamento através de atributos privados e métodos públicos (getters e setters), garantindo o controle adequado sobre quais dados podem ser modificados e como são acessados.

### Organização em Pacotes
O código está organizado em pacotes bem estruturados, facilitando a manutenção e reutilização do código.

### Herança
Utilizamos herança para reutilizar código entre classes relacionadas, permitindo que classes especializadas herdem comportamentos da classe pai.

### Sobrescrita de Métodos
Os métodos são sobrescritos nas subclasses para implementar comportamentos específicos de acordo com a demanda de cada classe especializada.

### Polimorfismo
O polimorfismo foi implementado utilizando referências a classes genéricas (superclasses) que se comportam de forma diferente dependendo da subclasse passada como parâmetro.

### Interfaces
Interfaces foram criadas e implementadas para definir contratos que as classes devem respeitar.

## Estrutura do Projeto

- **Modelos**: Classes que representam entidades do domínio (Filme, Série, etc.)
- **Interfaces**: Contratos que definem comportamentos esperados
- **Aplicação**: Classes responsáveis pela lógica principal da aplicação

## Desafio Final

O projeto inclui uma aplicação desenvolvida como desafio final do curso, integrando todos os conceitos de POO aprendidos.

## Aprendizados

Este projeto é um excelente exemplo prático de como aplicar os princípios de POO em Java, desde o planejamento da arquitetura até a implementação de classes, interfaces e relacionamentos entre objetos.
